**Goal:** Add an MDX-backed developer blog — a left-aligned "Writing" feed of
the 3 latest posts on the homepage (click-to-expand accordion with
syntax-highlighted code) plus a `/blog` archive of all posts — staying visually
true to the existing site and fully static-export-safe.

**Architecture:** Posts are MDX files in `content/posts/`, read + parsed at
build by a server-only loader (`src/posts`). MDX bodies are compiled to React at
build via `next-mdx-remote/rsc` + `rehype-pretty-code` (Shiki). A small client
`BackgroundGradient` (the animated gradient `<main>` + the existing
`setBackgroundStyles` ref) wraps both pages so `HomePage`/`BlogPage` can be
Server Components rendering the async `PostList`. Each `PostItem` is a client
accordion (CSS grid-row animation, `aria-expanded`, hash deep-link). Identity
(Header/Social) and Sanity wiring are unchanged.

**Tech Stack:** Next.js 16 (App Router, static export), React 19, TypeScript,
`next-mdx-remote`, `rehype-pretty-code`, `shiki`, `gray-matter`, CSS Modules,
Vitest, Yarn 4.

### Step 4: Implement

`src/posts/getPosts.ts`:

```ts
import { existsSync, readdirSync, readFileSync } from 'node:fs'
import { join } from 'node:path'
import matter from 'gray-matter'

import type { Post } from 'types'

const POSTS_DIR = join(process.cwd(), 'content', 'posts')

const formatDate = (date: string): string =>
  new Date(date).toLocaleDateString('en-US', {
    day: 'numeric',
    month: 'short',
    timeZone: 'UTC',
    year: 'numeric',
  })

export const getPosts = async (dir: string = POSTS_DIR): Promise<Post[]> => {
  if (!existsSync(dir)) {
    return []
  }

  const posts = readdirSync(dir)
    .filter((file) => file.endsWith('.mdx'))
    .map((file) => {
      const raw = readFileSync(join(dir, file), 'utf8')
      const { content, data } = matter(raw)
      const date = new Date(data.date).toISOString().slice(0, 10)

      return {
        body: content,
        date,
        formattedDate: formatDate(date),
        slug: file.replace(/\.mdx$/, ''),
        title: data.title as string,
      }
    })

  return posts.sort((a, b) => (a.date < b.date ? 1 : -1))
}
```

`src/posts/index.ts`:

```ts
export * from './getPosts'
```

Note: `gray-matter` parses the YAML `date` to a JS `Date`;
`new Date(data.date).toISOString().slice(0,10)` normalizes to `YYYY-MM-DD` for
stable string sorting and serialization. `async` keeps a uniform interface even
though reads are sync.

```mdx
AKT (Agent Knowledge Toolkit) gives coding agents something they usually lack: a
memory across sessions. Before starting a non-trivial task, an agent can
**recall** relevant past work; when a chunk of work wraps up, it can **capture**

- `akt start-story` / `akt finish-story` record a new decision with its

That "decision → because → rejected alternative" shape is the whole point: it's
```

```tsx
import { compileMDX } from 'next-mdx-remote/rsc'
import rehypePrettyCode from 'rehype-pretty-code'

import type { ReactNode } from 'react'

export const compilePostBody = async (body: string): Promise<ReactNode> => {
  const { content } = await compileMDX({
    options: {
      mdxOptions: {
        rehypePlugins: [
          [rehypePrettyCode, { keepBackground: false, theme: 'github-dark' }],
        ],
      },
    },
    source: body,
  })

  return content
}
```

(`keepBackground: false` lets the frosted `pre` panel from `PostItem.module.css`
show through while Shiki still colors the tokens inline.)

```tsx
import styles from './PostList.module.css'

interface PostListProps {
  posts: Post[]
}

export const PostList = async ({ posts }: PostListProps) => {
  const items = await Promise.all(
    posts.map(async ({ body, ...meta }) => ({
      content: await compilePostBody(body),
      meta,
    })),
  )

  return (
    <div className={styles.list}>
      {items.map(({ content, meta }) => (
        <PostItem key={meta.slug} {...meta}>
          {content}
        </PostItem>
      ))}
    </div>
  )
}
```
